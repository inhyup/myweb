Object oriented Programming

- 인간 중심적 프로그래밍

- 재사용이 높음(신뢰성 ↑) → 생산성↑

- overhead (Big-O) 발생, 하지만 하드웨어의 발전으로 보완

- 단점: object가 상태를 가짐 (예측할 수 없는 상태) → bug 발생

SOLID

Signle Responsibility Principle (SRP) - 하나의 책임만 있어야 함

Open-Closed Principle (OCP) - 확장에는 open, 변경에는 closed

Liskov Substitution Principle (LSP) - 상위 객체는 하위 객체로 바꿔도 정상 동작

Interface Segregation Principle (ISP) - 인터페이스는 분리

Dependency Inversion Principle (DIP) - 고수준 모듈은 저수준 모듈의 구현에 의존하지 않음

Encapsulation

프로그램의 세부 구현을 외부로 드러나지 않도록 특정 모듈(클래스) 내부로 감추는 것

public : 클래스의 외부에서 사용 가능하도록 노출

protected : 다른 클래스에게는 노출되지 않지만, 상속받은 자식 클래스에게는 노출

private : 클래스의 내부에서만 사용되며 외부로 노출되지 않음

Inheritance

자식 클래스가 부모 클래스의 특성과 기능을 그대로 물려받는 것

Overriding

클래스간 상속 관계에서 메서드를 재정의

부모 클래스에서는 virtual

자식 클래스에서는 override 구문을 매서드 앞에 정의

Polymorphism

하나의 변수명, 함수명 등이 상황에 따라 다른 의미로 해석될 수 있는 것

Overloading

매서드의 이름이 같고, 파라미터는 다르게

Method Signature: 메서드 이름 + 파라미터 정보

동일하거나 유사한 일을 수행하는 메서드가 전달받는 파라미터에 따라 조금씩 다른 연산을 해야 하는 경우
