RestfulAPI

API 설계의 중심에 Resource가 있고 HTTP Method를 통해 자원을 처리하도록 설계하는 방법

-Resource와 행위를 명시적이고 직관적으로 분리

Message는 Header와 Body를 명확하게 분리해서 사용

API 버젼 관리

서버와 클라이언트가 같은 방식을 사용해서 요청

장점

- Open API 제공 쉬움

- 멀티플랫폼 지원 및 연동 용의

- 원하는 타입으로 데이터를 주고 받음

- 기존 웹 인프라(HTTP)를 그대로 사용

단점

- 사용 가는 메소드가 4가지 밖에 없음

- 분산환경에 부적합

- HTTP 통신 모델에만 지원
